Este proyecto implementa un pipeline completo de an√°lisis geoespacial para la clasificaci√≥n autom√°tica de coberturas del suelo utilizando im√°genes satelitales Sentinel-2. La metodolog√≠a combina t√©cnicas de segmentaci√≥n de im√°genes (SLIC) con algoritmos de machine learning no supervisado (K-Means y Gaussian Mixture Models) para generar mapas de cobertura del suelo de alta precisi√≥n.

Objetivos: - Desarrollar un sistema automatizado de clasificaci√≥n de coberturas del suelo - Comparar diferentes t√©cnicas de segmentaci√≥n y clustering - Generar mapas tem√°ticos interpretables para an√°lisis territorial - Evaluar la separabilidad espectral de diferentes clases de cobertura

1. Contexto y Problema

1.1 Motivaci√≥n

La clasificaci√≥n de cobertura del suelo es fundamental para m√∫ltiples aplicaciones: - Planificaci√≥n urbana y territorial: Identificaci√≥n de √°reas urbanas, agr√≠colas y naturales - Monitoreo ambiental: Seguimiento de cambios en vegetaci√≥n, cuerpos de agua y uso del suelo - Agricultura de precisi√≥n: Delimitaci√≥n de parcelas y tipos de cultivo - Gesti√≥n de recursos h√≠dricos: Identificaci√≥n y monitoreo de cuerpos de agua

1.2 Desaf√≠o T√©cnico

Las im√°genes satelitales contienen informaci√≥n espectral rica pero requieren procesamiento avanzado para extraer informaci√≥n sem√°ntica √∫til. Los desaf√≠os principales incluyen:

  • Variabilidad espectral: M√∫ltiples objetos pueden tener firmas espectrales similares

  • Escalabilidad: Procesamiento eficiente de im√°genes de alta resoluci√≥n
  • Interpretabilidad: Generar resultados que puedan ser validados y utilizados por expertos

1.3 Enfoque Propuesto

3. Metodolog√≠a

3.1 Pipeline de Procesamiento

El proceso de clasificaci√≥n sigue estos pasos:

  1. Preprocesamiento: Normalizaci√≥n y preparaci√≥n de bandas espectrales
  2. Segmentaci√≥n: Agrupaci√≥n de p√≠xeles espacialmente coherentes usando SLIC
  3. Extracci√≥n de caracter√≠sticas: C√°lculo de estad√≠sticas espectrales por segmento
  4. Clustering: Aplicaci√≥n de algoritmos no supervisados (K-Means, GMM)
  5. An√°lisis espectral: Validaci√≥n mediante firmas espectrales
  6. Post-procesamiento: Asignaci√≥n de clases sem√°nticas y generaci√≥n de mapas

3.2 Algoritmos Utilizados

Segmentaci√≥n SLIC (Simple Linear Iterative Clustering) - Agrupa p√≠xeles en ‚Äúsuperp√≠xeles‚Äù espacialmente coherentes - Reduce la dimensionalidad del problema (de ~1M p√≠xeles a ~300 segmentos) - Preserva bordes y estructuras espaciales

K-Means Clustering - Algoritmo de clustering particional - R√°pido y eficiente para grandes vol√∫menes de datos - Requiere especificar el n√∫mero de clusters (k)

Gaussian Mixture Models (GMM) - Modela distribuciones de probabilidad para cada clase - Permite asignaciones probabil√≠sticas (m√°s flexible que K-Means) - Mejor para capturar transiciones graduales entre clases

4.1.2 Optimizaci√≥n de Par√°metros de Segmentaci√≥n

Para seleccionar los mejores par√°metros, evaluamos m√∫ltiples configuraciones de SLIC comparando m√©tricas de calidad como el n√∫mero de segmentos, tama√±o promedio y variabilidad.

# ============================================================================
# FUNCIONES AUXILIARES PARA EVALUACI√ìN DE SEGMENTACI√ìN
# ============================================================================

def evaluate_segmentation_quality(segments, rgb_image):
    """
    Eval√∫a la calidad de una segmentaci√≥n usando m√©tricas b√°sicas.
    
    Par√°metros:
    -----------
    segments : array
        Matriz de segmentos etiquetados
    rgb_image : array
        Imagen RGB normalizada
        
    Retorna:
    --------
    dict : Diccionario con m√©tricas de calidad
    """
    n_segments = segments.max()
    
    # Calcular estad√≠sticas de tama√±o de segmentos
    segment_sizes = np.bincount(segments.flatten())
    segment_sizes = segment_sizes[segment_sizes > 0]  # Remover segmentos vac√≠os
    
    metrics = {
        'n_segments': n_segments,
        'avg_size': np.mean(segment_sizes),
        'std_size': np.std(segment_sizes),
        'min_size': np.min(segment_sizes),
        'max_size': np.max(segment_sizes),
        'size_ratio': np.max(segment_sizes) / np.min(segment_sizes) if np.min(segment_sizes) > 0 else float('inf')
    }
    
    return metrics

def compare_segmentations(segmentations_dict, rgb_image, title="Comparaci√≥n de Segmentaciones"):

    Compara m√∫ltiples configuraciones de segmentaci√≥n visual y num√©ricamente.

    plt.tight_layout()
    plt.show()

4.2 Clasificaci√≥n No Supervisada

Una vez segmentada la imagen, aplicamos algoritmos de clustering para agrupar segmentos con caracter√≠sticas espectrales similares. Esto nos permite identificar diferentes tipos de cobertura del suelo sin necesidad de datos de entrenamiento etiquetados.

4.2.1 Determinaci√≥n del N√∫mero √ìptimo de Clusters

Antes de aplicar K-Means, necesitamos determinar cu√°ntos clusters (clases) son apropiados para nuestros datos. Utilizamos el M√©todo del Codo (Elbow Method) para identificar el punto √≥ptimo.

# ============================================================================
# M√âTODO DEL CODO PARA DETERMINAR N√öMERO √ìPTIMO DE CLUSTERS
# ============================================================================

# El m√©todo del codo busca el punto donde agregar m√°s clusters no mejora significativamente
# la reducci√≥n de inercia (suma de distancias al cuadrado dentro de cada cluster)

print("üîç Aplicando m√©todo del codo para determinar k √≥ptimo...")

# Preparar datos (usar caracter√≠sticas espectrales de segmentos)
X = media_por_segmento

# Probar diferentes valores de k
k_values = range(2, 11)
inertias = []

for k in k_values:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    kmeans.fit(X)
    inertias.append(kmeans.inertia_)

# Visualizar curva del codo
fig, ax = plt.subplots(figsize=(10, 6))

4.2.3 Clasificaci√≥n con Gaussian Mixture Models (GMM)

Como alternativa a K-Means, implementamos Gaussian Mixture Models que modelan las distribuciones de probabilidad de cada clase. GMM es especialmente √∫til para capturar transiciones graduales entre clases y manejar mejor la variabilidad espectral.

# ============================================================================
# CLASIFICACI√ìN CON GAUSSIAN MIXTURE MODELS (GMM)
# ============================================================================

# GMM con caracter√≠sticas espectrales + textura para mejor separaci√≥n
# Esto permite distinguir mejor entre clases espectralmente similares (ej: urbano vs suelo)

from scipy.ndimage import median_filter, uniform_filter


print("üîÑ Aplicando Gaussian Mixture Models...")

H, W, C = rgb_scaled.shape
img8 = (rgb_scaled * 255).astype(np.uint8)

# Extraer caracter√≠sticas espectrales en m√∫ltiples espacios de color
lab = cv2.cvtColor(img8, cv2.COLOR_RGB2LAB).astype(np.float32)
hsv = cv2.cvtColor(img8, cv2.COLOR_RGB2HSV).astype(np.float32)
L, a, b = [lab[..., i] for i in range(3)]
S, V = hsv[..., 1], hsv[..., 2]

# Extraer caracter√≠sticas de textura (√∫til para separar urbano de suelo)
gray = cv2.cvtColor(img8, cv2.COLOR_RGB2GRAY).astype(np.float32)
win = 7
mean = uniform_filter(gray, size=win)
mean2 = uniform_filter(gray**2, size=win)
std_local = np.sqrt(np.clip(mean2 - mean**2, 0, None))
edges = cv2.Canny(img8, 60, 120)
edge_density = uniform_filter(edges.astype(np.float32), size=win)

# Combinar todas las caracter√≠sticas
R, G, B = [rgb_scaled[..., i] for i in range(3)]
features = np.dstack([
    R, G, B,                    # RGB original
    L, a, b,                    # Espacio LAB
    S / 255.0, V / 255.0,       # Espacio HSV
    std_local / (std_local.max() + 1e-6),      # Textura: desviaci√≥n est√°ndar local
    edge_density / (edge_density.max() + 1e-6)  # Textura: densidad de bordes
]).astype(np.float32)

# Preparar datos para GMM
X = features.reshape(-1, features.shape[2])
mask_valid = np.isfinite(X).all(axis=1)
Xv = X[mask_valid]

# Estandarizar caracter√≠sticas
scaler = StandardScaler()
Xv_std = scaler.fit_transform(Xv)

# Aplicar GMM con k=5
k_gmm = 5
gmm = GaussianMixture(
    n_components=k_gmm,
    covariance_type='tied',      # Compartir matriz de covarianza (m√°s eficiente)
    init_params='kmeans',        # Inicializar con K-Means
    random_state=42,
    reg_covar=1e-6
)
gmm.fit(Xv_std)

# Predecir etiquetas
labels_full = np.full(X.shape[0], -1, dtype=int)
labels_full[mask_valid] = gmm.predict(Xv_std)
labels_img = labels_full.reshape(H, W)

# Suavizado espacial para reducir ruido
labels_smooth = median_filter(labels_img, size=3)

# Visualizar resultado
fig, axes = plt.subplots(1, 2, figsize=(16, 8))

axes[0].imshow(rgb_scaled)
axes[0].set_title('Imagen Original', fontsize=12, fontweight='bold')
axes[0].axis('off')

im = axes[1].imshow(labels_smooth, cmap='tab10', interpolation='nearest')
axes[1].set_title(f'Clasificaci√≥n GMM (k={k_gmm}, features espectrales + textura)', 
                  fontsize=12, fontweight='bold')
axes[1].axis('off')

cbar = plt.colorbar(im, ax=axes[1], fraction=0.046, pad=0.04)
cbar.set_label('Cluster ID', rotation=270, labelpad=15)

plt.tight_layout()
plt.show()

print("‚úÖ Clasificaci√≥n GMM completada")
print("   Ventaja: Mejor separaci√≥n de clases espectralmente similares mediante textura")
